# 🎮 织梦者 · 体验指引

> 本文档面向评委与其他访客，说明如何直接在浏览器中体验游戏，无需安装任何环境。

---

## 🚀 三步开始游戏

### 第 1 步：Fork 仓库

点击仓库右上角的 **Fork** 按钮，将项目复制到你的个人空间。

### 第 2 步：点击橙色「预览」按钮

进入 Fork 后的仓库页面，点击右上角的 **橙色按钮**。

- 按钮文字通常为「预览」或「Preview」
- 点击后**无需进入 IDE 开发环境**，会直接跳转到游戏预览页面

### 第 3 步：等待加载，开始游玩

预览页面启动后，浏览器会加载 Godot Web 导出产物（约几十秒，取决于网络）。看到游戏标题画面即可开始体验。

---

## ⏱ 预期等待时间

| 阶段 | 预计耗时 | 说明 |
|------|----------|------|
| 容器启动 | 10~20 秒 | CNB 拉起预览容器 |
| 资源加载 | 20~60 秒 | 下载 `.wasm` + `.pck` 资源包 |
| 总计 | 约 1 分钟内 | 加载完成后自动进入标题界面 |

> 容器保活时长为 **1 小时**（3600000 毫秒），期间可反复刷新体验；超时后重新点击橙色按钮即可再次启动。

## ❓ 常见问题

### Q1：点击橙色按钮后进入了代码编辑器（IDE），而不是游戏？

这不符合预期。仅预览模式已开启（`onlyPreview: true`），应直接进入游戏页面。若进入 IDE，请确认 Fork 的是最新 `main` 分支，并刷新页面后重试。

### Q2：页面加载后白屏或卡住？

Godot 4 Web 导出依赖 `SharedArrayBuffer`，需要浏览器支持跨域隔离头。请使用 **Chrome / Edge 最新版**，并确认未禁用 JavaScript。加载 `.pck` 资源包较大，首次加载请耐心等待。

### Q3：提示镜像拉取失败 / 容器启动失败？

镜像地址为公开常量地址 `docker.cnb.cool/copyworms/hachathondreamweaver:latest`，制品库已设为公开。若仍失败，可能是 CNB 平台临时波动，请稍后重试。

### Q4：游戏玩到一半突然断开？

容器保活时长为 1 小时，超时会自动停止。重新点击橙色按钮即可重新启动一局。

## 📦 技术栈速览

- **引擎**：Godot 4.6（GL Compatibility Mode）
- **导出**：Web Export（WASM + SharedArrayBuffer）
- **部署**：Docker + Nginx（Alpine，含 COOP/COEP 跨域隔离头）
- **平台**：CNB（cnb.cool）仅预览模式
